# 团建助手 - 事件风暴输出文档集 (Team Building Assistant - Event Storming Outputs)

## 项目概述

本项目采用**领域驱动设计(DDD)**方法论，通过**事件风暴**工作坊的形式，深度分析并设计了团建助手系统。整个项目由7个不同角色的专家共同参与，产出了一套完整的企业级团建管理解决方案。

## 系统架构总览

### 6个限界上下文

1. **活动规划上下文** (核心域)
   - AI驱动的活动推荐
   - 个性化活动策划
   - 活动评估和优化

2. **预算管理上下文** (支撑域)
   - 预算编制和审批
   - 成本跟踪
   - 财务报告

3. **团队协调上下文** (支撑域)
   - 成员管理
   - 时间协调
   - 分组管理

4. **反馈分析上下文** (支撑域)
   - 反馈收集
   - 数据分析
   - 改进建议

5. **媒体分享上下文** (通用域)
   - 照片视频管理
   - 权限控制
   - 分享功能

6. **日程管理上下文** (通用域)
   - 时间安排
   - 日历集成
   - 提醒功能

### 技术选型

**后端**: Java 17 + Spring Boot 3.2
**前端**: React 18 + Redux Toolkit
**数据库**: PostgreSQL + MongoDB + Redis
**消息队列**: Apache Kafka
**部署**: Kubernetes + Docker
**监控**: Prometheus + Grafana
